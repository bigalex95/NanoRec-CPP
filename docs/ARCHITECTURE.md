## Class Responsibilities

### ğŸ¯ Application Class

**Purpose**: Manages the entire application lifecycle  
**Responsibilities**:

- Initialize all subsystems
- Run the main application loop
- Clean up resources on shutdown
- Coordinate between different components

### ğŸ“Š Version Class

**Purpose**: Centralized version and build information  
**Responsibilities**:

- Store version numbers (MAJOR.MINOR.PATCH)
- Display version information
- Show dependency versions (ImGui, etc.)

### ğŸ“ Logger Class

**Purpose**: Unified logging across the application  
**Responsibilities**:

- Timestamp all log entries
- Support multiple log levels
- Format log output consistently
- (Future) Write logs to file

### âš™ï¸ Config Class

**Purpose**: Manage application configuration  
**Responsibilities**:

- Store video/audio/app settings
- Load configuration from file
- Save configuration to file
- Provide defaults
